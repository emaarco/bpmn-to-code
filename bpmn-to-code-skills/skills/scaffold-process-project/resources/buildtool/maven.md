# Maven Build Configuration Templates

Use this file when `buildTool=MAVEN`. Read the relevant section for the chosen engine and
approach, substitute `{{placeholders}}`, and apply to `pom.xml`.

The Spring Boot parent BOM (generated by Spring Initializr) already manages H2 and
PostgreSQL driver versions — do not add explicit versions for those.

---

## 1. bpmn-to-code plugin (add inside `<build><plugins>`)

Look up the latest version at https://central.sonatype.com/artifact/io.github.emaarco/bpmn-to-code-maven

For Java projects change `src/main/kotlin` to `src/main/java` and `KOTLIN` to `JAVA`.

```xml
<plugin>
    <groupId>io.github.emaarco</groupId>
    <artifactId>bpmn-to-code-maven</artifactId>
    <version>{{bpmnToCodeVersion}}</version>
    <executions>
        <execution>
            <goals>
                <goal>generate-bpmn-api</goal>
            </goals>
        </execution>
    </executions>
    <configuration>
        <baseDir>${project.basedir}</baseDir>
        <filePattern>src/main/resources/bpmn/**/*.bpmn</filePattern>
        <outputFolderPath>${project.basedir}/src/main/kotlin</outputFolderPath>
        <packagePath>{{packageName}}.api</packagePath>
        <outputLanguage>KOTLIN</outputLanguage>
        <processEngine>{{engine}}</processEngine>
        <useVersioning>false</useVersioning>
    </configuration>
</plugin>
```

## 3. Dependencies (add inside `<dependencies>`)

### CAMUNDA_7 — process-engine-api (default)

```xml
<!-- process-engine-api: engine-neutral abstraction layer -->
<dependency>
    <groupId>dev.bpm-crafters.process-engine-adapters</groupId>
    <artifactId>process-engine-adapter-camunda-platform-c7-embedded-spring-boot-starter</artifactId>
</dependency>
<dependency>
    <groupId>dev.bpm-crafters.process-engine-api</groupId>
    <artifactId>process-engine-api</artifactId>
    <version>1.5</version>
</dependency>
<dependency>
    <groupId>dev.bpm-crafters.process-engine-worker</groupId>
    <artifactId>process-engine-worker-spring-boot-starter</artifactId>
    <version>0.8.0</version>
</dependency>
<!-- Camunda 7 engine -->
<dependency>
    <groupId>org.camunda.bpm.springboot</groupId>
    <artifactId>camunda-bpm-spring-boot-starter-webapp</artifactId>
    <version>7.24.0</version>
</dependency>
<!-- Database drivers (versions managed by Spring Boot parent) -->
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
<dependency>
    <groupId>org.postgresql</groupId>
    <artifactId>postgresql</artifactId>
    <scope>runtime</scope>
</dependency>
```

## 4. Codegen command

```bash
mvn io.github.emaarco:bpmn-to-code-maven:generate-bpmn-api
```

## 5. Build command

```bash
mvn verify
```

## 6. Run commands

```bash
# Local profile (H2, no Docker — embedded engines only)
mvn spring-boot:run -Dspring-boot.run.profiles=local

# Full stack (PostgreSQL via Docker)
mvn spring-boot:run
```
